# OfferMind 产品核心结构

---

## 1️⃣ Problem（痛点）

求职者在面试过程中积累了大量经验，但这些经验是**碎片化、主观化、不可复用的**。

- **面试完就忘**：每轮面试的问题、反馈、表现细节随时间迅速消散，无法形成系统性记忆
- **无法量化成长**：候选人凭"感觉"判断自己是否进步，缺少客观的能力追踪机制
- **复盘流于表面**：传统笔记式复盘（"感觉还行"/"答得不好"）缺乏结构化分析，无法指导下一轮准备
- **跨岗位经验断裂**：面 Google 和面字节的经验无法互相迁移，每次都从零开始
- **决策盲区**：当多个 Offer 同时到来时，候选人缺乏基于自身成长数据的理性决策框架

> 市面上的求职工具解决的是 **"状态追踪"** 问题（投了哪些公司、到了哪一轮）。  
> 但没有人解决 **"认知自己"** 的问题——我到底在哪些能力上真正成长了？

---

## 2️⃣ Insight（洞察）

**面试不应该是一次性事件，而应该是可复用的职业数据资产。**

每一场面试都包含丰富的信号：
- 面试官关注了哪些能力维度？
- 我在哪些维度表现稳定，哪些反复暴露短板？
- 跨公司、跨岗位的能力趋势是什么？

这些信号如果被结构化提取、跨时间聚合、智能分析，就能从"一堆面试经历"变成**个人职业成长的数据引擎**。

核心洞察：
> 求职不是一个 **追踪问题**，而是一个 **决策质量问题**。  
> 帮候选人做出不会后悔的职业决策，比帮他们通过下一轮面试更有价值。

---

## 3️⃣ Solution（产品）

OfferMind 是一个 **AI 原生的面试智能系统**，通过三层 AI Agent 将面试经历转化为结构化的职业智能。

### Layer 1：Interview Analysis Agent（单轮面试解析）

- **输入**：面试记录 / 转录文本 / 手写笔记（支持中英双语）
- **处理**：AI 自动提取问题、分类题型与难度、评估回答质量
- **输出**：10 维能力评分（产品思维、执行力、沟通等）+ 逐题 SWOT 分析 + 结构化反思

> 将模糊的面试印象转化为**可量化、可对比的数据记录**。

### Layer 2：Role Debrief Agent（岗位聚合分析）

- **输入**：同一岗位的 2-6 轮面试数据
- **处理**：多轮信号聚合，面试官画像识别，能力热力图生成
- **输出**：
  - 面试官关注维度映射表
  - 10 维能力热力图（跨轮次加权）
  - 录用概率预测（Low / Medium / High + 置信度 + 理由）
  - 关键洞察（公司最关心什么 / 你的优势 / 你的风险）
  - 下一步最佳行动建议

> 从"每轮单独看"升级为**整个岗位的全局视角**。

### Layer 3：Career Growth Intelligence Agent（职业成长智能）

- **输入**：跨公司、跨岗位的所有面试数据（时间线）
- **处理**：纵向能力演化分析，转折点检测，频率-影响优先级排序
- **输出**：
  - 10 维能力的时间演化曲线
  - 成长转折点识别（哪次面试让你能力跃迁？）
  - 持续短板 vs 临时波动的区分
  - 个性化成长路线图

> 让候选人看到自己的**职业成长轨迹**，而不只是一堆孤立的面试结果。

### 辅助层：Career Signal Timeline（职业信号时间线）

- AI 过滤噪音活动，只保留真正影响职业轨迹的关键信号
- 模式识别：跨时间段、跨角色的行为模式
- 叙事解读：不只告诉你发生了什么，还告诉你**为什么重要**

### 基础设施：Job Dashboard（求职看板）

- 可视化 Kanban 看板追踪所有申请状态
- 多管线支持（秋招 / 春招 / 社招可独立追踪）
- 拖拽操作，支持面试阶段自定义
- 与 AI 分析层深度集成

## 4️⃣ Your Role（我的角色）

**独立全栈开发者 · 从 0 到 1 · 一人完成所有角色。**

| 角色 | 具体工作 |
|------|---------|
| **产品经理** | 定义三层 AI 架构、设计 10 维能力框架、规划用户旅程、撰写完整 PRD |
| **AI 系统设计师** | 设计多 Agent 协作架构、定义严格的 JSON Schema 输出规范、实现证据驱动评分机制（每个分数必须附带原文证据，减少 AI 幻觉） |
| **前端工程师** | React 18 + TypeScript + Tailwind CSS + shadcn/ui，实现雷达图、热力图、时间线等复杂数据可视化 |
| **后端工程师** | Supabase（PostgreSQL + Auth + Edge Functions），设计多管线数据模型，实现 RLS 安全策略 |
| **Prompt 工程师** | 为 6 个 AI Agent 设计结构化 Prompt，确保跨语言（中英双语）输出一致性和 Schema 合规性 |
| **用户体验设计师** | 深色科技风 UI 设计，移动端适配，拖拽交互，动效设计 |

### 技术决策亮点

- **选择 Gemini 3 Flash Preview** 作为统一模型：长上下文推理（处理多轮面试历史）+ 严格结构化输出（复杂 JSON Schema）+ 多模态输入（PDF/图片解析），一个模型覆盖所有 Agent 需求
- **Evidence-Based Scoring 机制**：所有 AI 评分必须引用原文证据，从架构层面解决 AI 幻觉问题
- **Pattern-over-Event 逻辑**：系统优先识别跨时间模式，避免因单次面试表现过度反应

### Learnings

1. **用户信任 AI 的前提是可解释性**  
   早期版本的 AI 输出虽然"看起来专业"，但用户不信任没有依据的评分。引入 Evidence-Based Scoring 后，用户接受度显著提升——每个分数背后都有原文引用，AI 不再是黑盒。

2. **长期价值来自模式识别，而非单次反馈**  
   单轮面试分析的价值是有上限的。真正让用户产生"啊哈时刻"的是跨岗位的能力趋势——发现自己在产品思维上持续进步，但在系统设计上始终原地踏步。这种纵向洞察是传统工具无法提供的。

3. **最好的 AI 系统增强人类判断力，而不是替代它**  
   OfferMind 不承诺"用了就能拿 Offer"，而是帮用户建立**决策框架**——基于数据而非焦虑做出职业选择。这种诚实定位反而赢得了更深的用户信任。

4. **结构化输出是 AI 产品的生命线**  
   6 个 Agent 的所有输出都遵循严格的 JSON Schema。这不仅保证了可视化层的稳定性，更让整个系统具备了"数据可组合性"——Layer 1 的输出可以直接喂给 Layer 2，Layer 2 喂给 Layer 3，形成真正的智能管线。

5. **Vibe Coding 的本质是产品思维驱动开发**  
   作为一个 AI PM 背景的独立开发者，我用"vibe coding"的方式快速迭代——不是先写代码再想产品，而是先想清楚用户需要什么洞察，再反推系统架构。这种 PM-first 的开发方式让产品始终围绕用户价值展开。
